Files

main.py Entry Point

Loads .env, initialises the bot via build_app(), starts Telegram long-polling.

load_dotenv run_polling
bot.py Telegram

All Telegram handlers. Every handler checks _is_authorized(). Long-running calls go through asyncio.to_thread() with a typing indicator.

/start /clear /vault /whisper handle_photo handle_message
llm.py Orchestration

Ollama API client. Runs a tool call loop (max 5 iterations). Three conversation modes: text, vision, one-shot (vault summary / tweet).

run() run_with_image() summarize_to_vault() whisper_tweet() tool loop
context.py State

In-memory conversation history. deque(maxlen=20) — oldest message dropped when full. Stores user, assistant, tool_calls, and tool result messages.

deque maxlen=20 get_history() clear()
vector.py Search

ChromaDB persistent client + Ollama embedding client. Chunks vault content (1500 chars / 150 overlap), indexes via embeddings, supports semantic search and random chunk retrieval.

index_file() search_vault_semantic() random_chunk() ChromaDB
tools.py Tools

Vault file I/O, git sync, website analysis. Exports TOOL_DEFINITIONS (OpenAI schema) and TOOL_HANDLERS (name → callable) consumed by llm.py.

read_vault write_vault sync_vault search_vault_semantic analyze_website
ingest.py Script

One-time bulk indexer. Scans VAULT_PATH recursively for *.md files and calls vector.index_file() for each. Run once at setup or after bulk vault changes.

python ingest.py bulk index

Data Flows

Text Message

1
Telegram TEXT → bot.handle_message()
2
llm.run(user_text) — adds to context, builds [system + history]
3
POST /v1/chat/completions with TOOL_DEFINITIONS
4
If tool_calls: execute via TOOL_HANDLERS, store result, loop (max 5×)
5
On stop: store assistant reply in context → Telegram reply

Photo Message (with optional caption)

1
Telegram PHOTO → bot.handle_photo()
2
Select largest photo size (photo[-1]), get_file() + download_as_bytearray()
3
base64.b64encode(bytes) → image_b64. Caption or "What is in this image?"
4
llm.run_with_image(caption, image_b64) — filters tool/tool_calls from history (Ollama vision limitation)
5
Builds [image_url content block + text], runs same tool loop as text flow
6
Edits "Processing image…" message with LLM reply

/vault Command

1
bot.cmd_vault() → context.get_history()[-10:]
2
llm.summarize_to_vault(recent) → returns {titel, samenvatting} JSON
3
Generate filename: YYYY-MM-DD_HH-MM_{titel}.md
4
tools.write_vault(filename, markdown) → writes file + auto-indexes in ChromaDB
5
tools.sync_vault() → git add . && git commit && git push

/whisper Command

1
vector.random_chunk() — random offset into ChromaDB collection
2
llm.whisper_tweet(chunk) — one-shot call, no tools, no history
3
English tweet ≤200 chars with 2–3 hashtags → Telegram reply

Vault Write → ChromaDB Index

1
tools.write_vault(file_name, content)
2
Path traversal check → os.makedirs → write file
3
vector.index_file(file_name, content)
4
Delete old chunks for file from ChromaDB
5
Chunk content (1500 chars / 150 overlap) → embed each via Ollama → store in ChromaDB

Tools Available to LLM

Tool Arguments Returns Notes
read_vault file_name: str File contents or error Path traversal protected
write_vault file_name: str, content: str Success or error Auto-creates dirs. Triggers ChromaDB indexing.
sync_vault (none) Git output or error git add . && commit && push. 30s timeout.
search_vault_semantic query: str Top 5 chunks with file names Embeds query via Ollama, queries ChromaDB
analyze_website url: str Extracted Markdown ≤8000 chars or error httpx + trafilatura. Rotating UA. HTTP/HTTPS only.

Security

  • Every Telegram handler checks ALLOWED_USER_ID before processing — all other users silently ignored.
  • Vault file access uses os.path.realpath comparison to prevent path traversal attacks.
  • analyze_website validates URL scheme — only http:// and https:// allowed (SSRF protection).
  • Tool call loop capped at 5 iterations to prevent infinite loops.
  • Vision requests (run_with_image) strip tool_calls and tool messages from history to avoid Ollama multimodal compatibility issues.
